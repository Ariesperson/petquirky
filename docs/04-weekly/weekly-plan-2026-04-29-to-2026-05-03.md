# PetQuirky Weekly Plan: 2026-04-29 to 2026-05-03

> Week: 2026-04-29 to 2026-05-03  
> PM owner: Project Lead / PM  
> Current phase: launch readiness, technical closure, product validation, content preparation

## This Week's Main Goal

Move PetQuirky from "core development mostly complete" to "ready for pre-launch QA review".

This week should not focus on traffic growth first. The priority is to make sure the purchase, payment, order, account, and email flow is reliable enough before bringing users into the store.

## Current Project Status

- Next.js 16.2.1, React 19, Tailwind CSS 4 storefront.
- `npm run lint` passes.
- `npm run build` passes.
- `npm run test:unit` currently has 3 failing tests.
- Current staged changes include account, order detail, PayPal capture, order email, i18n, and related tests.
- Product catalog currently has 6 products and 5 categories.
- `README.md` now references `docs/01-launch/launch-checklist.md`.

## Daily Execution Plan

### Wednesday, 2026-04-29: Technical Closure

**Owner:** Technical Development Engineer

Focus on the 3 failing unit tests:

- `tests/unit/email.test.ts`
- `tests/unit/order-detail-client.test.tsx`
- `tests/unit/paypal-button.test.tsx`

Expected output:

- Fixes for failing behavior or outdated tests.
- Short explanation of what changed.
- Test results for:
  - `npm run lint`
  - `npm run test:unit`
  - `npm run build`

Done means:

- All three verification commands pass, or all remaining blockers are documented with exact failing tests and suspected causes.

### Thursday, 2026-04-30: Main Flow QA

**Owner:** QA Launch Checker

Verify the core purchase journey:

- Browse `/en`, `/de`, `/fr`, `/es`.
- Product listing and product detail pages.
- Cart and checkout.
- PayPal sandbox order creation and capture.
- Checkout success page.
- Supabase order persistence.
- Account order list and order detail page.
- Customer and admin email notification.

Expected output:

- Manual QA notes.
- Blocking, high-priority, and deferrable issue list.
- `docs/01-launch/launch-checklist.md`.

Done means:

- The checklist exists and can be reused before every release.
- Any issue that blocks a real order is clearly marked as blocking.

### Friday, 2026-05-01: Compliance and Trust Review

**Owner:** Compliance and Customer Experience

Review customer-facing trust and policy areas:

- Private-seller identity.
- 14-day return policy.
- Shipping and delivery expectations.
- Refund and return limitations.
- Privacy policy and cookie notice.
- Contact method and expected response time.
- PayPal trust messaging.
- Order confirmation and notification email wording.

Expected output:

- Compliance/trust issue list.
- Suggested wording changes where needed.
- Clear notes for anything that requires human legal confirmation.

Done means:

- No page or email makes unrealistic claims.
- The site clearly communicates seller identity, returns, shipping, and support expectations.

### Saturday, 2026-05-02: Product and Conversion Review

**Owners:** Product and Supply Chain Analyst, Conversion and Ecommerce Operations

Review the current 6-product catalog.

Scoring dimensions:

- TikTok demonstration potential.
- Margin potential.
- Supply stability.
- European demand fit.
- Differentiation.
- Logistics and after-sales risk.
- Website content potential.

Conversion review should cover:

- Product page value proposition.
- Product specs and use cases.
- Suitable pet type.
- Trust modules.
- Bundle or upsell ideas.
- Free-shipping threshold ideas.

Expected output:

- Product scoring table.
- Keep / revise / replace recommendation for each product.
- Product page copy and module improvement list.

Done means:

- The first product batch is no longer judged by taste alone.
- Each product has a clear next action.

### Sunday, 2026-05-03: Content Preparation and Weekly Review

**Owners:** TikTok Content Operator, SEO Content Operator, Review and Data Analyst

Create initial content plans based on the strongest products and categories.

TikTok output:

- 7 scripts.
- Each script includes:
  - Hook.
  - Shot order.
  - On-screen subtitles.
  - Voiceover or no-voiceover recommendation.
  - CTA.
  - Hashtags.
  - Validation hypothesis.

SEO output:

- 10 article topics.
- First 3 outlines.
- Each article includes:
  - Search intent.
  - Title.
  - Meta description.
  - H1/H2 structure.
  - Internal links.
  - Product recommendation slots.

Weekly review output:

- What is now launch-ready.
- What still blocks launch.
- What should move into next week's plan.
- Stop / continue / double-down recommendations.

Done means:

- Next week can start with either launch cleanup or first product/content execution, not another round of vague planning.
